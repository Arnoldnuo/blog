# 自动化监控系统规划

## 目标

1. 好用易用
   1. 开发中可以随时调用系统进行测试，可以指定机器域名进行测试。
   2. 上线前进行自动测试，无需额外操作。（提交代码自动触发case）
   3. case生成的便利性。
   4. case运行失败后，失败原因人性化输出。（指名什么页面，什么功能报错）
2. 稳定性
3. 监控功能
   1. 监控结果输出
   2. ​



1. ​



### 一个用户使用的完整功能

1. 手写case&录制case
2. 开发中，一条命令跑case，输出指定测试机的测试结果
3. 上线前跑全量case，预览机上也可以跑全量case

## 需要完善的功能

1. 随时调用监控系统的功能
2. 可以指定任意机器域名进行测试的功能。
3. 真机测试插件化，和phantomjs并列
4. 真机测试结果输出
